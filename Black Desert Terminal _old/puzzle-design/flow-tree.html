BRE NETWORK — DATABASE UNLOCK FLOW TREE

Puzzle progression map. Start: BRE-06 welcome screen → Final: BRE-01 Brandt log. Branching paths with mixed clue types.

// OVERVIEW

Players begin at BRE-06. The welcome screen hands them the first password (the BRE-06 intranet). Inside that intranet, the first lore clue points at the next database. Each database either contains a plain password, a lore-based hint, or a composite clue requiring info from two databases.

Site progression is broadly: BRE-06 → BRE-05 → (BRE-04 + BRE-03 in parallel) → BRE-02 → BRE-01 (final). The final database (Brandt's personal log at BRE-01) requires a synthesized password using clues from BRE-03 and BRE-01.

GREEN — given to player on welcome screen
BLUE — site intranet (default per-site password)
YELLOW — internal database (puzzle-locked)
ORANGE — requires combining 2 clues
RED — final climactic reveal

// DESIGN NOTES

Clue-type variety per edge

  • Plain handoff (4 edges): previous DB literally prints the next password. Used for site→site transitions to avoid dead-ends.
  • Lore-keyword (8 edges): the next password is a name/term highlighted in the current DB (Orpheus, CARE-9, Null-Slate, Kess-C12, Malek-440, Hollowed, Black-Glass, Denver-2079).
  • Cipher / composite (2 edges): the BRE-04 Confidential Archive needs MNEMO+SYNE concatenation; the final Brandt log needs SILENT+ARRAY combined across sites.

Failsafes (so a player can never fully soft-lock)

  • After 3 failed password attempts on a DB, the terminal should print: "Hint: cross-reference the highlighted entity name."
  • The "Operator" can be a recurring fallback NPC who answers a single radio question per site (in-fiction reason for soft-hint).
  • BRE-03 must be marked as required — consider a Personnel Registry note that explicitly says "Brandt's clearance was issued through ridge-line observation. Without ridge logs his archive cannot be addressed."

New databases I'm proposing (1 net-new)

  • BRE-03 // Metaplanar Observations — slot exists in your folder (BRE-03 metaplanar_observations_de.md) but wasn't part of my deep read. If it's already drafted, just align its password to DIS-RIM and add the closing memo to Shadow Spirits.
  • All other databases reuse what you already have — only the passwords and a few closing memo lines need updating.

What stays / what changes

  • Stays: all your existing entries, lore, German narrative voice, file structure.
  • Changes: the password: frontmatter field on each DB (placeholders like database1 become lore-driven), plus one new closing memo per DB acting as the handoff clue.
  • If you'd prefer to keep the original passwords (labor, schatten, etc.), the structure still works — just the clues need to point at the kept words.

// PLAYER FLOW SUMMARY (one paragraph)

The player connects to BRE-06. The welcome screen prints the intranet password and a one-line operator note hinting at "Orpheus." Inside, the BRE-06 intranet hands them ORPHEUS through a lab-lock notice. Inside the Research Lab, they meet CARE-9 by name. Inside the CARE-9 reports, the AI's final evaluation gives them a plain handoff to BRE-05 (ECHO-4429). From BRE-05 they follow Null-Slate the courier into the Personnel Registry, which gives them two site passwords at once — BRE-04 and BRE-03. They puzzle through BRE-04's split security databases (combining MNEMO + SYNE for the Confidential Archive) and the BRE-03 side-trip (which whispers half a codename: SILENT). BRE-04 hands them BRE-02. BRE-02 hands them BRE-01. Inside BRE-01 they unlock the Ares discovery records and the executive pressure chain — which gives them the second half: ARRAY. Putting SILENT and ARRAY together, they finally crack open Brandt's personal log and read what really happened at BRE-07.
