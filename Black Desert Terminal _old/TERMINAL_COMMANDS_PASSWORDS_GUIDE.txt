BLACK DESERT RESEARCH TERMINAL
Commands, Passwords, and First-Time User Guide

WARNING FOR GAME MASTERS
This file contains hidden commands and passwords. Do not hand this file to players
unless you want them to know the secrets.


============================================================
1. FIRST-TIME USER GUIDE
============================================================

What this terminal is:
The terminal is a roleplay interface for exploring Black Desert / ARES data.
Players can load databases, search logs, open diagnostics, and inspect facility
status screens.

Best way to open it:
Use the GitHub Pages URL. If testing locally, use a local web server instead of
opening index.html directly with file://. Some browser features, database loading,
ZIP loading, and content fetches work best over HTTP.

Basic controls:
- Wait for the boot screen, or press the skip button.
- You can type commands directly. You do not need to click the command bar first.
- Press Enter to run a command.
- Arrow Up / Arrow Down in the command bar recalls command history.
- Arrow Up / Arrow Down outside the command bar moves through the left menu.
- Enter outside the command bar runs the selected left-menu item.
- Escape closes most popups or returns from full-page tool screens.

Slash prefix:
- All commands now use a leading slash, e.g. /HELP, /SEARCH, /CONNECT.
- The legacy unprefixed form (HELP, SEARCH, ...) is still accepted by the parser
  as an alias for backward compatibility, but everything the terminal prints
  (menu labels, help text, hints, autocomplete) shows the slash form.

First useful commands:
1. /HELP
   Shows the in-terminal help screen.

2. /LOAD DATABASE
   Opens the database selector. Pick a database, then enter that database password.

3. /SEARCH <term>
   Example: /SEARCH profit
   Searches safe metadata: topic, date, and keywords.

4. /ACCESS
   Opens the access password prompt. Use this for Elevated or Admin clearance.

5. /FSEARCH <term>
   Example: /FSEARCH anomaly
   Requires Elevated or Admin clearance. Searches deeper data, including message text.

6. /DIAGNOSTIC
   Opens the Base Diagnostic screen.

7. /TOPOGRAPHY
   Opens the Black Desert tactical map / facility screen.
   (Formerly FACILITY STATUS. Old aliases still work.)

Search explained simply:
- SEARCH is safe and available to everyone.
- SEARCH checks Topic, Date, and Keywords.
- SEARCH does not search inside the full message text.
- FSEARCH is deeper and requires Elevated or Admin access.
- FSEARCH checks Topic, ID or Person, Date, Keywords, and Message content.
- If your clearance is too low, an entry can appear but the message text may be redacted.

Clearance levels:
- Employee: normal user. Shown in cyan.
- Elevated: unlocks FSEARCH and some confidential files. Shown in amber.
- Admin: full access and admin commands. Shown in red.

Database loading:
- The terminal can mount up to 3 databases at once.
- Use LOAD DATABASE for databases from the /databases folder.
- Use LOAD FILE for a local .md, .txt, .dat, or ZIP database package.
- Database passwords are the roleplay passwords shown later in this file.
- ZIP packages use a separate hidden transport key. Players should not need to type it.

Returning from tools:
- DIAGNOSTIC and TOPOGRAPHY open as full-page screens.
- Use [ BACK TO TERMINAL ] to return.
- The terminal restores the previous session without replaying the boot sequence.

Top bar buttons:
- SND ON / SND OFF toggles sound.
- NET ONLINE / NET OFFLINE toggles network systems.
- FX changes visual effects mode.
- SYS ONLINE starts terminal shutdown. Use Reboot Terminal on the offline screen to boot again.

If something looks slow:
- Use FX LOW or PERFORMANCE ON.
- Use SAFE MODE for a stronger low-effects mode.
- If NET is offline, Diagnostic and Topography are intentionally unavailable.

============================================================
7. PLAYER-FACING QUICK START
============================================================

If you want a short player version, use this:

1. Press the boot skip button if you do not want to wait.
2. Type /HELP if you are lost.
3. Type /LOAD DATABASE and pick a database.
4. Enter the password you were given for that database.
5. Type /SEARCH followed by one word, for example /SEARCH profit.
6. If you receive higher clearance, type /ACCESS and enter the new password.
7. Use /FSEARCH after gaining Elevated or Admin access.
8. Type /DIAGNOSTIC or /TOPOGRAPHY to inspect live systems.
9. Use [ BACK TO TERMINAL ] to return from full-screen tools.
10. If systems are unavailable, check whether NET is OFFLINE.


============================================================
8. COMMON PROBLEMS
============================================================

Problem:
I can see an entry but the message is redacted.

Reason:
Your clearance is too low for that entry.

Fix:
Use /ACCESS and enter an Elevated or Admin password.


Problem:
FSEARCH does not work.

Reason:
/FSEARCH requires Elevated or Admin access.

Fix:
Use /ACCESS first.


Problem:
Diagnostic or Topography will not open.

Reason:
NET is probably offline.

Fix:
Use /NET ON or click the NET status button.


Problem:
The terminal looks slow or laggy.

Fix:
Use /FX LOW, /PERFORMANCE ON, or /SAFE MODE.


Problem:
Database loading does not work when opening index.html directly.

Reason:
Some browser features are restricted in file:// mode.

Fix:
Use GitHub Pages or start a local web server.
